# Скрипт отправки выгрузки субсидий

## Что делает
Скрипт:
1. Берёт корневую папку `source_dir`.
2. Проверяет все вложенные подпапки.
3. Валидной считает подпапку, где есть **хотя бы один `.dbf`** и **хотя бы один `.xls`**.
4. Все валидные подпапки складывает в **один zip-архив**.
5. Отправляет архив на основной адрес.
6. После успешной отправки удаляет только валидные отправленные папки.
7. Если есть невалидные папки или не удалось удалить уже отправленные папки — пишет в лог и отправляет письмо об ошибке.

## Что считается ошибкой
- подпапка есть, но в ней нет `.dbf`
- подпапка есть, но в ней нет `.xls`
- подпапка есть, но нет обоих файлов
- письмо не отправилось
- архив не создался
- отправленная папка не удалилась

## Файлы
- `subsidy_mailer.py` — основной скрипт
- `config.example.json` — пример настроек

## Запуск
```bash
python subsidy_mailer.py config.json
```

Если путь к конфигу не передать, скрипт попробует открыть `config.json` в текущей папке.

## Пример структуры
```text
source_dir/
├── ЖЭУ_1/
│   ├── data.dbf
│   └── report.xls
├── ЖЭУ_2/
│   ├── data.dbf
│   └── report.xls
└── ЖЭУ_3/
    └── only_one_file.xls
```

В этом примере `ЖЭУ_1` и `ЖЭУ_2` попадут в архив, а `ЖЭУ_3` попадёт в лог ошибок.

## Планировщик
Если нужна периодичность, скрипт лучше запускать через:
- **Планировщик заданий Windows**
- либо **cron** в Linux

Саму периодичность удобнее не вшивать в код, а задавать через планировщик.

## Что можно быстро доработать
- проверка ровно двух файлов, а не "есть хотя бы один `.dbf` и один `.xls`"
- поддержка `.xlsx`
- отдельная папка `processed/` вместо удаления
- HTML-письма
- несколько разных правил отправки через таблицу/JSON
