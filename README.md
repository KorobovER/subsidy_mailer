## Описание

**Subsidy Mailer** - это автоматизированное приложение для сбора, архивации и отправки файлов субсидий по электронной почте. Приложение сканирует указанную директорию на наличие папок с файлами `.dbf` и `.xls`, создает архивы с датированной структурой и отправляет их по email.

## Инфраструктура

### Серверы

- **Сервер приложения**: Debian 9 (IP: 10.10.20.250)
  - На этом сервере установлен и работает скрипт `subsidy_mailer.py`
  - Отсюда выполняются все операции: сканирование, архивация, отправка email
  - Настроен автоматический запуск через Cron (08:00 и 18:00)

- **Файловый сервер**: Windows Server 2003 (IP: 10.10.30.62)
  - Хранит исходные файлы для обработки
  - Путь к исходным файлам: `C:\base\CES\!Выгрузка\Субсидии`
  - Путь для архивов: `C:\base\CES\!Выгрузка\Архив\Субсидии`
  - Доступ через сетевое монтирование Samba/CIFS

### Сетевое взаимодействие

```
Debian 9 (10.10.20.250)           Windows Server 2003 (10.10.30.62)
┌─────────────────────────┐       ┌─────────────────────────────┐
│ subsidy_mailer.py       │       │ C:\base\CES\!Выгрузка\      │
│ Python 3.5              │◄──────►│ Субсидии/ (исходные файлы)   │
│ SMTP: mail.vc.local     │       │ Архив/Субсидии/ (архивы)    │
│ Cron: 08:00, 18:00      │       │ Samba/CIFS монтирование      │
└─────────────────────────┘       └─────────────────────────────┘
```

### Монтирование сетевых папок

На сервере приложения (Debian 9) сетевые папки смонтированы:

```bash
/mnt/old_server/ → //10.10.30.62/base/CES/!Выгрузка/
```

- `/mnt/old_server/Субсидии` - исходные файлы для обработки
- `/mnt/old_server/Архив/Субсидии` - папка для сохранения архивов

## Функциональность

### Основные возможности

- **Автоматическое сканирование** директории с файлами субсидий
- **Валидация файлов** - проверка наличия ровно одного файла `.dbf` и одного файла `.xls` в каждой папке
- **Создание архивов** с организацией по дате (YYYYMMDD)
- **Отправка email** с архивом в виде вложения
- **Удаление обработанных папок** после успешной отправки
- **Отчет об ошибках** по email при возникновении проблем
- **Логирование** всех операций

### Обработка ошибок

Приложение отслеживает и сообщает о следующих типах ошибок:

- **Невалидные папки** - папки с неправильным количеством или типом файлов
- **Файлы вне папок** - файлы `.dbf` и `.xls`, находящиеся непосредственно в исходной директории
- **Ошибки удаления** - папки, которые не удалось удалить после отправки

### Параметры конфигурации

#### Директории
- **source_dir** - Путь к директории с файлами для обработки
- **archive_dir** - Путь к директории для сохранения архивов
- **log_file** - Путь к файлу логов

#### Настройки обработки
- **delete_sent_folders** - Удалять ли папки после успешной отправки (true/false)

#### Параметры SMTP
- **smtp_host** - Адрес SMTP сервера
- **smtp_port** - Порт SMTP сервера (обычно 25, 587, 465)
- **smtp_username** - Имя пользователя для SMTP аутентификации
- **smtp_password** - Пароль для SMTP аутентификации
- **use_tls** - Использовать TLS шифрование (true/false)

#### Параметры email
- **from_email** - Email адрес отправителя
- **to_email** - Список получателей (массив строк)
- **cc_email** - Список адресов для копии (массив строк)
- **error_email** - Список адресов для отчетов об ошибках (массив строк)
- **subject** - Тема письма
- **body** - Тело письма

## Процесс работы

### 1. Инициализация

1. Загрузка конфигурации из JSON файла
2. Настройка логирования
3. Проверка доступности исходной директории

### 2. Сканирование директории

Приложение сканирует `source_dir` и классифицирует содержимое:

- **Валидные папки** - содержат ровно один файл `.dbf` и один файл `.xls`
- **Невалидные папки** - содержат неправильное количество или тип файлов
- **Файлы вне папок** - файлы `.dbf` и `.xls` в корневой директории

### 3. Создание архива

При наличии валидных папок:

1. Создается директория с текущей датой: `archive_dir/YYYYMMDD/`
2. Создается временная папка с временем: `subsidies_HHMMSS/`
3. Копируются все валидные папки во временную директорию
4. Создается ZIP архив из временной директории
5. Временная директория удаляется

### 4. Отправка email

1. Формируется email сообщение с темой и телом из конфигурации
2. ZIP архив добавляется как вложение
3. Отправляется email всем получателям и адресам в копии
4. Логируется успешная отправка

### 5. Удаление обработанных папок

Если в конфигурации указано `delete_sent_folders: true`:

1. Проверяется возможность удаления каждой папки
2. Успешно удаленные папки логируются
3. Ошибки удаления записываются в отчет

### 6. Обработка ошибок

При наличии ошибок:

1. Формируется отчет с детальным описанием всех проблем
2. Отправляется email на адреса из `error_email`
3. Логируется информация об ошибках

## Структура архивов

Архивы организуются по следующей структуре:

```
archive_dir/
├── 20260325/                  # Папка с датой (YYYYMMDD)
│   ├── subsidies_080000.zip   # Архив с временем (HHMMSS)
│   └── subsidies_180000.zip
├── 20260326/
│   └── subsidies_080000.zip
└── ...
```

Каждый ZIP архив содержит папки с исходными файлами:

```
subsidies_080000.zip
├── test/
│   ├── file.dbf
│   └── file.xls
└── test2/
    ├── data.dbf
    └── data.xls
```

## Развертывание

### Требования к системе

- **ОС**: Debian 9 или совместимый Linux
- **Python**: 3.5+
- **Пакеты**: cifs-utils (для монтирования сетевых папок)

### Сетевые требования

- Доступ к SMTP серверу (mail.vc.local:25)
- Доступ к сетевой папке (10.10.30.62)
- Права на монтирование сетевых ресурсов

### Процесс развертывания

1. **Установка зависимостей**
   ```bash
   sudo apt-get update
   sudo apt-get install -y python3 python3-pip cifs-utils
   ```

2. **Создание директории**
   ```bash
   sudo mkdir -p /opt/subsidy_mailer
   cd /opt/subsidy_mailer
   ```

3. **Копирование файлов**
   ```bash
   scp subsidy_mailer.py root@10.10.20.250:/opt/subsidy_mailer/
   scp config-prod.json root@10.10.20.250:/opt/subsidy_mailer/
   ```

4. **Монтирование сетевой папки**
   ```bash
   sudo mkdir -p /mnt/old_server
   sudo mount -t cifs //10.10.30.62/base/CES/\!Выгрузка \
     -o username=администратор,password=qqq,iocharset=utf8 \
     /mnt/old_server
   ```

5. **Настройка Cron**
   ```bash
   (crontab -l 2>/dev/null; echo "0 8 * * * cd /opt/subsidy_mailer && python3 subsidy_mailer.py config-prod.json >> /opt/subsidy_mailer/cron.log 2>&1"; echo "0 18 * * * cd /opt/subsidy_mailer && python3 subsidy_mailer.py config-prod.json >> /opt/subsidy_mailer/cron.log 2>&1") | crontab -
   ```

## Запуск и использование

### Ручной запуск

Для немедленной отправки файлов без ожидания расписания:

```bash
cd /opt/subsidy_mailer
python3 subsidy_mailer.py config-prod.json
```

**Когда использовать ручной запуск:**
- Срочная отправка файлов вне расписания
- Тестирование после изменений конфигурации
- Проверка работы после решения проблем

## Устранение неполадок

### Частые проблемы

#### 1. Ошибка "Source directory not found"
**Причина**: Сетевая папка не смонтирована
**Решение**: Проверьте монтирование и смонтируйте заново

#### 2. Ошибка SMTP "Connection unexpectedly closed"
**Причина**: Проблемы с подключением к SMTP серверу
**Решение**: Проверьте доступность сервера и параметры аутентификации

#### 3. Ошибка "Permission denied"
**Причина**: Недостаточно прав на запись/удаление
**Решение**: Проверьте права доступа к директориям

#### 4. Ошибка "No valid folders found"
**Причина**: В директории нет папок с правильной структурой файлов
**Решение**: Проверьте наличие папок с файлами .dbf и .xls
